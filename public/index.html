1. Hugoとは

1.1 Hugoの概要

HugoはGo言語で書かれた静的サイトジェネレーターです。マークダウンファイルからHTMLサイトを生成し、高速なビルドと豊富な機能が特徴です。

Hugoの特徴:
  • 超高速ビルド: 数千ページでも数秒でビルド
  • プラグイン不要: 必要な機能がすべて内蔵
  • テーマシステム: 豊富なテーマとカスタマイズ性
  • 多言語対応: 複数言語サイトの構築が簡単
  • ホットリロード: 開発中のリアルタイム更新

1.2 静的サイトの利点

セキュリティ

  • データベースがないためSQLインジェクション等の脅威がない
  • サーバーサイドの処理がないため攻撃面が少ない

パフォーマンス

  • 事前生成されたHTMLファイルを配信するため高速
  • CDNとの相性が抜群
  • サーバー負荷が非常に軽い

コスト効率

  • 静的ファイルホスティングサービス（GitHub Pages、Netlifyなど）で無料運用可能
  • サーバー管理コストが不要

2. インストールと初期設定

2.1 Hugoのインストール

Windows

# Chocolateyを使用
choco install hugo-extended

# Scoopを使用
scoop install hugo-extended

macOS

# Homebrewを使用（推奨）
brew install hugo

Linux (Ubuntu/Debian)

# Snap（推奨）
sudo snap install hugo

# 手動インストール（最新版）
wget https://github.com/gohugoio/hugo/releases/download/v0.xxx.x/hugo_extended_0.xxx.x_Linux-64bit.tar.gz
重要: hugo-extended版をインストールしてください。SCSS/SASSの処理に必要です。

2.2 新しいサイトの作成

# 新しいサイトを作成
hugo new site my-awesome-site

# 作成されたディレクトリに移動
cd my-awesome-site

# 開発サーバーを起動
hugo server

3. contentフォルダ詳細解説

3.1 コンテンツの基本構造

content/
├── _index.md           # ホームページ
├── about.md            # Aboutページ
├── contact.md          # Contactページ
├── posts/              # ブログ記事セクション
│   ├── _index.md       # 記事一覧ページ
│   ├── first-post.md
│   └── second-post.md
├── docs/               # ドキュメントセクション
│   ├── _index.md
│   ├── getting-started.md
│   └── advanced/
│       ├── _index.md
│       └── configuration.md
└── projects/           # プロジェクトセクション
    ├── _index.md
    ├── project-1.md
    └── project-2.md

3.2 ページタイプの理解

Single Pages（個別ページ）

  • about.md → /about/
  • posts/first-post.md → /posts/first-post/

List Pages（一覧ページ）

  • posts/_index.md → /posts/（記事一覧）
  • docs/_index.md → /docs/（ドキュメント一覧）

4.2 ベーステンプレート（baseof.html）

<!DOCTYPE html>
<html lang="{{ .Site.Language.Lang }}">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>
        {{- if .IsHome }}
            {{ .Site.Title }}
        {{- else }}
            {{ .Title }} | {{ .Site.Title }}
        {{- end }}
    </title>
    
    {{- partial "head.html" . }}
</head>
<body>
    {{- partial "header.html" . }}
    
    <main>
        {{- block "main" . }}
        <!-- デフォルトのコンテンツ -->
        {{- end }}
    </main>
    
    {{- partial "footer.html" . }}
</body>
</html>

5. テンプレート入門

5.1 Go Templateの基本文法

変数の出力

<!-- 基本的な変数出力 -->
{{ .Title }}
{{ .Content }}
{{ .Date }}

<!-- エスケープなしの出力（HTMLを含む場合） -->
{{ .Content | safeHTML }}

<!-- パイプ処理 -->
{{ .Title | upper }}
{{ .Date.Format "2006-01-02" }}
{{ .Content | truncate 100 }}

条件分岐

<!-- if文 -->
{{- if .Params.featured }}
<div class="featured-post">特集記事</div>
{{- end }}

<!-- if-else文 -->
{{- if .Params.author }}
<span class="author">{{ .Params.author }}</span>
{{- else }}
<span class="author">Anonymous</span>
{{- end }}

ループ処理

<!-- 基本のrange -->
{{- range .Pages }}
<h2><a href="{{ .RelPermalink }}">{{ .Title }}</a></h2>
{{- end }}

<!-- 条件付きrange -->
{{- range where .Pages "Type" "post" }}
<article>{{ .Title }}</article>
{{- end }}

6. フロントマター完全ガイド

6.1 フロントマターの基本

フロントマターはコンテンツファイルの先頭に記述するメタデータです。YAML、TOML、JSONの形式で書けます。

YAML形式（推奨）

---
title: "Hugoの完全ガイド"
date: 2023-12-01T10:00:00+09:00
lastmod: 2023-12-15T15:30:00+09:00
draft: false
author: "田中太郎"
description: "Hugoの使い方を初心者にも分かりやすく解説します"
tags: ["Hugo", "静的サイト", "Web開発"]
categories: ["チュートリアル"]
featured_image: "/images/hugo-guide.jpg"
---

6.2 標準フロントマター変数

必須・基本変数

---
title: "記事のタイトル"                    # ページタイトル（必須）
date: 2023-12-01T10:00:00+09:00           # 作成日時（必須）
draft: false                              # 下書きフラグ
---

SEO関連

---
title: "ページタイトル"
description: "ページの説明文。検索結果に表示されます。"
keywords: ["キーワード1", "キーワード2"]
slug: "custom-url-slug"                   # URLのカスタマイズ
url: "/custom/path/"                      # 完全なURLパス指定
aliases: ["/old-url/", "/another-old-url/"] # リダイレクト設定
---

9. Hugo Pipesとアセット管理

9.1 Hugo Pipesの概要

Hugo Pipesは、CSS、JavaScript、画像などのアセットを処理・最適化するHugo内蔵の機能です。

基本的な使用方法

<!-- layouts/partials/head.html -->
{{- $style := resources.Get "scss/main.scss" | toCSS | minify | fingerprint }}
<link rel="stylesheet" href="{{ $style.RelPermalink }}" integrity="{{ $style.Data.Integrity }}">

{{- $script := resources.Get "js/main.js" | minify | fingerprint }}
<script src="{{ $script.RelPermalink }}" integrity="{{ $script.Data.Integrity }}"></script>

9.2 画像処理

<!-- 画像のリサイズと最適化 -->
{{- $image := resources.Get "images/hero.jpg" }}
{{- $resized := $image.Resize "800x600 q85" }}
<img src="{{ $resized.RelPermalink }}" alt="Hero image" width="800" height="600">

10. ショートコード活用術

10.1 ショートコードの基本

ショートコードは、マークダウンの中で再利用可能なHTMLコンポーネントを呼び出す仕組みです。

基本的な使用方法

<!-- パラメーター付きショートコード -->
{{< youtube "dQw4w9WgXcQ" >}}

<!-- ペアショートコード -->
{{< highlight javascript >}}
function hello() {
  console.log("Hello, World!");
}
{{< /highlight >}}

<!-- 名前付きパラメーター -->
{{< figure src="image.jpg" title="画像のタイトル" caption="キャプション" >}}

10.2 カスタムショートコードの作成

<!-- layouts/shortcodes/button.html -->
{{- $url := .Get "url" | default "#" }}
{{- $text := .Get "text" | default "ボタン" }}
{{- $class := .Get "class" | default "btn btn-primary" }}

<a href="{{ $url }}" class="{{ $class }}">
    {{ $text }}
</a>

12. SEO最適化

12.1 基本的なSEO設定

<!-- layouts/partials/seo.html -->
{{- $title := .Title }}
{{- $description := .Description | default .Summary | default .Site.Params.description }}

<title>
{{- if .IsHome }}
    {{ .Site.Title }} | {{ .Site.Params.description }}
{{- else }}
    {{ $title }} | {{ .Site.Title }}
{{- end }}
</title>

<meta name="description" content="{{ $description | truncate 160 }}">
<link rel="canonical" href="{{ .Permalink }}">

<!-- Open Graph -->
<meta property="og:type" content="{{ if .IsPage }}article{{ else }}website{{ end }}">
<meta property="og:title" content="{{ $title }}">
<meta property="og:description" content="{{ $description | truncate 300 }}">

13. デプロイとホスティング

13.1 GitHub Pagesデプロイ

# .github/workflows/gh-pages.yml
name: GitHub Pages

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
        with:
          submodules: true
          fetch-depth: 0

      - name: Setup Hugo
        uses: peaceiris/actions-hugo@v2
        with:
          hugo-version: '0.120.0'
          extended: true

      - name: Build
        run: hugo --minify

      - name: Deploy
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./public

13.2 Netlifyデプロイ

# netlify.toml
[build]
  publish = "public"
  command = "hugo --gc --minify"

[build.environment]
  HUGO_VERSION = "0.120.0"
  HUGO_ENV = "production"

14. パフォーマンス最適化

14.1 ビルド最適化

# hugo.toml
[build]
  writeStats = true
  
[caches]
  [caches.images]
    dir = ":resourceDir/_gen"
    maxAge = "24h"
  [caches.assets]
    dir = ":resourceDir/_gen"
    maxAge = "24h"

[minify]
  disableCSS = false
  disableHTML = false
  disableJS = false

14.2 画像最適化

<!-- レスポンシブ画像 -->
{{- $image := resources.Get "images/photo.jpg" }}
{{- $small := $image.Resize "400x q85" }}
{{- $medium := $image.Resize "800x q85" }}
{{- $large := $image.Resize "1200x q85" }}

    <source media="(max-width: 599px)" srcset="{{ $small.RelPermalink }}">
    <source media="(max-width: 999px)" srcset="{{ $medium.RelPermalink }}">

15. トラブルシューティング

# テンプレートの構文チェック
